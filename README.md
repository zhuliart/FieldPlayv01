# Handoff:FieldPlay 田游智耕 · 正式版（全量真实数据）

> 交付给 Claude Code 用于在真实代码库中完成全量开发。本目录内的 HTML 文件是**设计参考原型**（展示外观与交互意图），**不是可直接上线的生产代码**。任务是：在目标工程的既有技术栈中（建议 React + TypeScript + 状态管理 + 后端 API；若无既有环境则自选最合适框架）**重建这套设计**，并把原型里的**模拟数据/本地模拟逻辑替换为真实数据与后端服务**。

## 1. 概述（Overview）

FieldPlay（田游智耕 · 铁娃子果蔬园）是一款**农场经营互动游戏**，核心体验：

- 一块 16:9 的等距农田，12 个地块，可种植 4 种作物（番茄 / 生菜 / 玉米 / 辣椒），每种 5 个生长阶段。
- **真实天气驱动**的昼夜循环与极端天气（连续暴雨 / 持续干旱 / 寒潮霜冻）影响作物。
- 两种玩法模式：**手动模式**（玩家自己种植、浇水、施肥、收获、除草、卖货）与**机器人托管模式**（AI 自主经营，自带强化学习）。
- 农业机器人沿**可编辑巡田路网**行动；商店买种/卖货、仓库囤货保值、市场行情实时波动、闲置地块课税等经济系统。

设计基调：明亮、卡通、拟物（厚描边木牌 / 圆角卡片 / 投影按钮），中文优先。

**当前原型 = 单机模拟**：经济行情、AI 学习、好友/订单/排行均为前端模拟，存档走 `localStorage`。**正式版需要后端、账号体系与真实数据**（见 §9）。

## 2. 保真度（Fidelity）

**高保真（hifi）**。原型即最终视觉：配色、字体、间距、圆角、投影、动效都已定稿，应**像素级还原**。原型的画面以 **1280×720 的固定设计舞台**为基准，整体等比缩放铺满视口（见 §4）。

## 4. 舞台与缩放（Stage & Scaling）

- 根容器 `#fp-wrap`：`position:fixed; inset:0;` 居中，深底 letterbox。
- 设计舞台 `#fp-root`：固定 `1280×720`，`transform-origin:center`，由 `fitBoard()` 按视口缩放（`scale=min(W/1280,H/720)`）。
- 移动端竖屏（触摸设备且视口竖向）自动 `rotate(90deg)` 横屏适配，`scale=min(W/720,H/1280)`。
- 所有 UI 用**绝对定位 + 舞台坐标**，不要用响应式流式布局——这是固定画布游戏。
- **重要**：所有浮层（toast、节点编辑、路径手柄）必须挂在 `#fp-root` 内部，才与缩放坐标系一致（原型曾因 toast/按钮挂在 `#fp-wrap` 外导致线上位置漂移）。

## 6. 交互与行为（Interactions & Behavior）

### 手动模式
- 选工具 → 点地块执行：浇水/施肥/收获/除草/清枯/耕地/覆膜/开沟/修路。
- 种植规划：选作物 → 点空地放种植点（拖动调整、单击修改、橡皮删除）→「完成」定植后开始生长与缺水检测。
- 移栽已生长作物有**枯死概率**（越成熟越高）。
- 浇水/施肥有粒子动效（水花环 + 水滴 / 生长爆发光 + 颗粒）。

### 托管（AI）模式
- 机器人沿路网自动巡田：播种/浇水/施肥/除草/收获/买种/卖货/囤货/充电。
- 自带轻量强化学习：每种作物的偏好 Q 值、探索率 `explore`、出售阈值 `sellThreshold`、囤货倾向 `storeBias`、磨损 `wear`、破产计数 `fails`；亏损/腐损/手续费/课税会反向调整策略；资金跌破 −20000 触发破产并重置资金（保留学习记录）。

### 巡田路径编辑
- 进入编辑模式若路网为空 → 自动按田间道路**种子化**路网。
- 节点：**点选**（变橙）、**拖动**（pointer capture）、**点两节点连线/断线**、空白处点击加节点（含就近吸附）、删除选中节点、清空、保存。

### 时间 / 天气
- 游戏时钟 = 农场当地时间（`Asia/Shanghai`），随 tick 同步；背景按时段交叉淡入。
- 真实天气每 10 分钟拉取一次，灾害天气有「初起→高潮→尾声」强度曲线，影响作物与任务。

### 经济
- 市场行情每 tick 波动（含暴涨/暴跌事件，toast 播报）。
- 商店买种（批量）/卖货；仓库囤货保值但有腐损与手续费；闲置地块超 `IDLE_LIMIT(45 ticks≈117s)` 触发轮转课税。

### 全屏 / 重置
- 全屏（iOS 提示横屏替代）；重置二次确认后清空全部 `localStorage` 存档。

## 7. 状态与数据模型（State Model）

顶层 state（原型在前端，正式版应由**后端 + 客户端状态**承载）：

```
mode            'manual' | 'auto'
level, xp, xpMax
coins, energy, energyMax, water, eco          // 资源
plots[12]       { id, slots[], tilled, harvested }
  slot          { x, y, crop, stage(0-4), growth, watered, needsWater,
                  dry, dead, committed, moist, age }
market / marketPrev   { tomato, lettuce, corn, chili }  // 价格倍率
ai              { funds, q{4作物}, trades, harvests, deaths, plantings,
                  explore, sells, decayLoss, feesPaid, idleTaxPaid,
                  spikeGain, sellThreshold, storeBias, wear, fails, last }
econ            { stock{4}, fresh, wh{4}, whBasis, decay, fee, seedStock }
robot           { battery, action, pos{left,top}, moveDur, moving, charging, module, face }
weather         { type, ticksLeft }            // type: clear/cloudy/lightrain/rain/drought/frost
tod             0..1（一天的归一化时刻）
geomOverride, roadNet, brush, tool, selected, modal, toast
```

### 关键常量（CROPS）
| 作物 | seed 成本 | sell 基价 | 5 阶段 |
|---|---|---|---|
| 番茄 tomato | 120 | 260 | 发芽→幼苗→开花→结果→成熟 |
| 生菜 lettuce | 80 | 180 | 发芽→幼苗→莲座→包心→成熟 |

### 存档（原型用 localStorage —— 正式版改后端账号持久化）
`fp_slots`（种植）、`fp_points`（种植点）、`fp_tilled`、`fp_harvested`、`fp_geom`（地块几何）、`fp_roadnet`（路网）、`fp_ai`（AI 学习）、`fp_econ`（经济）。

## 8. 性能注意（来自线上实测）

- **移动端内存**：背景预解码必须**只预热当前+过渡时段的 2–4 张**，禁止一次性预解码全部 36 张高清图（iOS Safari 会内存溢出崩溃）。生产实现同理：按需加载/解码背景，控制同屏解码态内存。
- **Modal 首开闪烁**：弹窗内容应在数据就绪后渲染（用就绪标记 / 骨架占位），避免露出未插值占位符。
- 事件绑定确保 `pointerdown/move/up` 正确映射（指针拖拽依赖之）。

## 9. ⭐ 模拟数据 → 真实数据映射（正式版核心改造）

| 原型中的模拟 | 正式版需要的真实数据/服务 |
|---|---|
| `localStorage` 存档 | 用户账号体系 + 后端持久化（农场、资金、作物、AI、路网均入库） |
| 市场行情前端随机波动 | 真实/服务端经济引擎（统一定价、防作弊、可运营调参），客户端只读 + 订阅推送 |
| AI 自主学习（前端 Q 值模拟） | 服务端托管经营逻辑或可信的本地策略 + 服务端校验结算；学习数据落库 |
| 好友 / 订单 / 排行（占位演示） | 真实社交关系链、订单系统、排行榜服务（分页、刷新、好友互动） |
| 等级 / 经验 / 资源初值（写死 68420 等） | 来自玩家账号的真实进度 |
| 真实天气（Open-Meteo 前端直连，经纬度写死 32.353,106.843 南江县） | 经服务端代理拉取并缓存（避免前端跨域/限流/坐标写死），支持多农场坐标 |
| 机器人行为（前端模拟） | 若对接真实农机/IoT，则消费真实设备状态与遥测；否则保持服务端权威模拟 |
| 资源消耗/收益结算在客户端 | 一律服务端权威结算，客户端乐观更新 + 校正 |

> 设计与交互**全部保留**；改造集中在「数据来源」与「结算权威」从前端模拟迁移到后端真实服务。
