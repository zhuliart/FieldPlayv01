# FieldPlay · DOM vs PixiJS(WebGL) 性能对比测量

> 对应任务书第六章「测量与对比协议」。**在同一部真机**上分别打开 DOM 原型（`FieldPlay.dc.html`）
> 与本 Pixi DEMO，三档各跑一遍，读右上角 FPS 表填表。建议覆盖 2–3 部机型（中端安卓 + iPhone）。

## 一、三档场景定义

| 档位 | 定义 | 本 DEMO 操作 |
|---|---|---|
| **静置** | 仅背景 + 静态地块，无生长/巡田/天气 | HUD →「静置」 |
| **常规** | tick 全开、机器人巡田、昼夜推进、偶发天气 | HUD →「常规」 |
| **压力** | 一键拉满：植株加密 + 连续天气切换 + 机器人巡田 + 粒子齐发 + 昼夜加速 | HUD →「压力」 |

> 切档会**自动清零最低 FPS**，停留 ~20–30s 待数值稳定后读「当前 / 均 / 低」三个值。

## 二、测量方法

1. 真机浏览器打开两版同一 URL（同分辨率、同亮度、关后台）。
2. 每档停留至读数稳定，记录 **当前/平均/最低 FPS**、**内存 MB**（Chrome 才有）、**首屏时间**（打开→可交互）。
3. DOM 原型无内建 FPS 表，可用 Chrome DevTools Performance / `Rendering → FPS meter`，或浏览器自带帧率工具。
4. Pixi 版直接读右上角表；可逐项关效果开关，定位各效果开销。

## 六、自测记录（本机·仅供管线验证，非真机结论）

- 环境：Playwright Chromium **无头 + SwiftShader 软件渲染（无 GPU）**，1280×720。
- 结果：启动无报错；WebGL 渲染器（type=WebGL）；作物图集合批成功（20 帧 → 1 base texture）；
  常规档 ~280 株、压力档 ~515 株；夜间车灯光池（color-dodge）、昼夜 tint、星空、浇水粒子、天气状态叠加均正常出帧。
- 帧率：软件渲染下偏低（属预期，CPU 软栅格化整屏混合），**真机 GPU 下不具可比性**，请以真机为准。
